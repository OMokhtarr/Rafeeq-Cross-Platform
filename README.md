# Rafeeq رفيق

A cross-platform Quran quiz and viewer app built with React, Ionic, and Capacitor. One codebase runs on **Web**, **iOS**, **Android**, and **Electron (desktop)**.

## Tech Stack

| Layer | Technology |
|---|---|
| UI Framework | [Ionic React 7](https://ionicframework.com/) + React 18 |
| Language | TypeScript 4.9 |
| Cross-Platform Bridge | [Capacitor 5](https://capacitorjs.com/) |
| Desktop (Electron) | `@capacitor-community/electron` |
| Routing | React Router 5 via `IonReactRouter` |

## How the App Works

### Startup & Data Seeding

On first launch the app seeds IndexedDB from a bundled `quran-text.json` file containing all 6,236 verses. This makes the app work fully **offline** after the first open. Fonts (QPC V4 Tajweed) are downloaded once from jsDelivr and cached in IndexedDB as blobs, so all 604 Quran pages render correctly without a network connection.

### Global State (Context API)

Four providers wrap the entire app in `App.tsx`:

| Context | What it manages |
|---|---|
| `ThemeContext` | Light/dark theme, persisted to localStorage |
| `LanguageContext` | Arabic/English UI language + RTL/LTR direction |
| `VerseVisibilityContext` | Which verse texts are hidden (used by quiz modes) |
| `PlaybackContext` | Audio queue, current reciter, playback controls |

### Navigation

React Router 5 routes are defined in `App.tsx`. A persistent `BottomNavBar` gives access to the five main sections: **Quran**, **Quiz**, **Azkar**, **Bookmarks**, and **Settings**.

Main routes:

```
/                                            → Home
/viewer                                      → Quran page viewer
/surah-juz                                   → Surah / Juz picker
/search                                      → Search
/azkar                                       → Azkar (daily remembrances)
/quiz-list                                   → Quiz selection
/akmel-alayah-setup  →  /akmel-alayah        → "Complete the verse" quiz
/mutashabihat-setup  →  /mutashabihat-test   → "Similar verses" quiz
/akmel-alnehayat-setup  →  /akmel-alnehayat  → "Beginning to ending" quiz
/account                                     → User account & goals
/bookmarks                                   → Saved verses
/settings                                    → App-wide settings
/playback                                    → Reciter & playback settings
/auth/callback                               → OAuth deep-link handler
```

### Audio Playback

`PlaybackContext` + the `usePlaybackQueue` hook manage verse-by-verse recitation. Audio files are downloaded as MP3 blobs, stored in IndexedDB, and served locally on subsequent plays. Over 200 reciters are supported.

### Authentication

OAuth 2.0 flow is handled by `oauth.service.ts`. On native platforms the callback is received via a deep link using the scheme `com.rafeeq.quranquiz://callback`. On web the browser redirects to `/auth/callback`.

## Running Locally

### Prerequisites

- Node.js 18+
- npm 9+

### Web (browser)

```bash
npm install
npm start
# Opens at http://localhost:3000
```

### Electron (desktop)

```bash
npm install
npm run electron:dev
# Starts the React dev server and opens the Electron window side-by-side
```

## Building for Each Platform

### Web production build

```bash
npm run build
# Output → /build
```

### iOS

> Requires macOS with Xcode 15+ and CocoaPods installed.

```bash
npm run ios:sync    # builds web assets and syncs to the ios/ Xcode project
npm run ios:open    # opens Xcode — run on simulator or device from there
```

Or run directly on a connected device/simulator:

```bash
npm run ios:run
```

### Android

> Requires Android Studio with the Android SDK.

```bash
npm run android:sync   # builds web assets and syncs to the android/ Gradle project
npm run android:open   # opens Android Studio
```

Or build a debug APK directly:

```bash
npm run android:build  # outputs to android/app/build/outputs/apk/debug/
```

### Electron (desktop)

```bash
npm run electron:build
# Packages the app for the current OS → electron/dist/
```

## CI/CD

A single GitHub Actions workflow (`.github/workflows/build-ios.yml`) runs on every push to `main` and on manual trigger.

**What it does:**
1. Installs Node 18 deps with `npm ci`
2. Builds the React web assets (`npm run build:prod`)
3. Syncs Capacitor to the `ios/` project (`npx cap sync ios`)
4. Installs CocoaPods
5. Imports the Apple signing certificate and provisioning profile from GitHub Secrets
6. Archives the app with Xcode and exports a signed `.ipa`
7. Uploads the `.ipa` as a GitHub Actions artifact (retained 30 days)

**Required GitHub Secrets:**

| Secret | Description |
|---|---|
| `BUILD_CERTIFICATE_BASE64` | Base64-encoded `.p12` distribution certificate |
| `P12_PASSWORD` | Password for the `.p12` file |
| `BUILD_PROVISION_PROFILE_BASE64` | Base64-encoded `.mobileprovision` file |
| `KEYCHAIN_PASSWORD` | Temporary keychain password used during the build |
| `APPLE_TEAM_ID` | Your Apple Developer Team ID |

## Key Concepts

### Offline-first

All Quran text, fonts, translations, and audio are cached locally in IndexedDB. The app is fully usable without internet after the first launch.

### Platform Detection

`platform.util.ts` detects whether the app is running as web, iOS, Android, or Electron and gates platform-specific behaviour (haptics, status bar, deep links) behind that check.

### IndexedDB Schema (v5)

| Store | Contents |
|---|---|
| `verses` | All 6,236 Quran verses (seeded at startup) |
| `pages` | API-fetched verse batches per page |
| `fonts` | QPC V4 Tajweed font blobs |
| `audio` | Recitation MP3 blobs |
| `translations` | Cached translation pages |
| `meta` | Migration flags |

### Capacitor App ID

`com.rafeeq.quranquiz` — used as the iOS bundle ID, Android application ID, and OAuth redirect scheme.
